[HTTP-1: 수업소개]
1990 - WEB을 공개
- 웹의 구성: HTML, URL, URI, Web browser, Web server, HTTP

* HTTP(HyperText Transfer Protocol)
- 점점 웹이 발달함에 따라 성능, 보안, 안정성 확보하기 어려워지고 있다.
- 현재는 텍스트 뿐만 아니라 이미지, 오디오, 동영상도 전송할 수 있게 되었다.
- 인터넷에서 가장 중요한 규칙

[HTTP-2: 소개]
* HTTP
- Request(Client): HTML을 주세요.
- Response(Server): HTML 여깄어요.

[HTTP-3: Request message]
* Request
- GET: 데이터를 웹 서버로부터 가져오는 메소드

[HTTP-4: Response message]
* Response
- version: HTTP 1.1
- status code: 404 not found, 403 forbidden

[HTTP-5: 수업을 마치며]

* HTTP vs HTTPS(SSL)
- HTTPS면 내용을 가져가더라도 암호화되어 있어 내용을 열 수 없다.

* Cache
- 저장된 파일을 읽어서 성능 향상
- cache-control, pragma

* Cookie
- 사용자를 식별
- Web storage로 업그레이드 되었다.

* Proxy server
- 중개 서버

* Wireshark
- 모든 네트워크 트래픽을 감시
- 무료, 복잡
